Open filter panel - Expected behavior

• Clicking "Filter Content" button slides Filter Content panel in from right (main page content slides left off screen).

• If no filters have been applied, "All types" and "All countries" are the default selections.

• If filters have been applied, the respective horizontal bar contains the names of the selected filters.

 

Expand Types panel - Expected Behavior

• Clicking "Types" bar slides "Select Types" panel in from bottom.

 

Expand Countries panel - Expected Behavior

• Clicking "Countries" bar slides "Select Countries" panel in from bottom.

 

Select Content Type - Expected Behavior

• The current selected filter is disabled (not clickable/hoverable).

• Clicking any active content type selects it. Try it with Apps or Sites (or All types).

• After selecting a content type, clicking "OK" slides the opened panel down, returning to "Select Types" panel.

• Clicking "OK" without selecting a new content type preserves the current selection as the active selection.

• Can only select one content type at a time.

 

Select Country - Expected Behavior

• Clicking any active country selects it. Try it with Country 1 (or All countries).

• After selecting a country, clicking "OK" slides the opened panel down, returning to "Select Countries" panel.

• Clicking "OK" without selecting a new country preserves the current selection as the active selection.

• Can only select one country at a time.

 

Back to Page - Expected Behavior

• Clicking "back" (arrow in upper left of Filter Content panel) applies the selected filters to the content on the screen where "Filter Content" panel was invoked.

• Clicking "back" arrow slides main page content (with filters applied) in from left ("Filter Content" panel slides right off screen).

How to use this prototype

• Clickable areas are indicated with a yellow hover state. This hover state is for your convenience only - there will be no hover state in the real experience.

• • You can see the effect of the filters by paying attention to whether or not a content block is an App or a Website, and whether or not the content block is Light Blue or Light Pink.

• The light blue content blocks are those attached to Country 1.

• The light pink content blocks are those attached to Country 2 (not actionable in this prototype).
