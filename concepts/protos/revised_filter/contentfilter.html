Open filter panel - Expected behavior

• Clicking "Filter Content" button slides Filter Content panel up from bottom (main page content remains below this panel).

• If no filters have been applied, "All types" and "Worldwide" are the default selections.

• If filters have been applied, the respective Type bar contains the names of the selected filters.

• Clicking "Cancel" closes Filter Content panel (slides down) and preserves existing selections on the screen where Filter Content panel was invoked (no changes applied).

• Clicking "Done" closes Filter Content Panel (slides down) and applies changed selections to the content on the screen where "Filter Content" panel was invoked.

 

Filter Content panel - Expected Behavior

• Clicking "Types" bar slides "Filter by Content Type" panel in from right (Filter Content panel slides left off screen).

 

Filter Content panel - Expected Behavior

• Clicking "Regional Relevance" bar slides "Filter by Regional Relevance" panel in from right (Filter Content panel slides left off screen).

 

Filter By Content Type panel - Expected Behavior

• The current selected filter is disabled (not clickable/hoverable).

• Clicking any active content type selects it. Try it with Apps or Webites (or All types).

• After selecting a content type, clicking "back" arrow in upper left slides the Filter By Content Type panel out to the right, bringing in Filter Content panel in from left.

• Clicking "back" arrow without selecting a new content type preserves the current selection as the active selection.

• Can only select one item type at a time (radio buttons).

 

• Filter By Regional Relevance panel - Expected Behavior

• The current selected filter is disabled (not clickable/hoverable).

• Clicking any active item selects it. Try it with Country 1 (or Worldwide).

• After selecting 1 or more options, clicking "back" arrow in upper left slides the Filter By Regional Relevance panel out to the right, bringing in Filter Content panel in from left.

• Clicking "back" arrow without selecting a new content type preserves the current selection as the active selection.

• Can select multiple options here, including Worldwide plus any number of countries.

• All countries is not an option. User can checkbox all the countries to achieve that if she wants to, though this seems a fringe use case that we should test post-launch (vs design/build now).

How to use this prototype

• Clickable areas are indicated with a yellow hover state. This hover state is for your convenience only - there will be no hover state in the real experience.

• You can see the effect of the filters by paying attention to whether or not a content block is an App or a Website, and whether or not the content block is Light Blue or Light Pink.

• The light blue content blocks are those attached to Country 1.

• The light pink content blocks are those attached to Worldwide.
